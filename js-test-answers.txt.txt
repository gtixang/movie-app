1.
В JavaScript распространение событий происходит в несколько этапов.
Когда событие возникает на каком-то элементе, оно проходит через три фазы:

Фаза погружения: Сначала событие "спускается" от корня DOM-дерева до целевого элемента.
Целевая фаза: Это момент, когда событие достигло элемента, на котором оно было вызвано.
Фаза всплытия: После того как событие "поймало" целевой элемент, оно начинает "всплывать" обратно,
проходя все родительские элементы до корня.

Например, с помощью event.stopPropagation() можно предотвратить всплытие или погружение события.

2.
Promise — это объект, который представляет результат асинхронной операции, которая может завершиться либо успешно,
либо с ошибкой.

Основные состояния:

Pending: Начальное состояние, ожидание.
Fulfilled: Когда асинхронная операция успешно завершена.
Rejected: Когда операция завершена с ошибкой.

Для обработки асинхронных операций также можно использовать async/await — синтаксический сахар над Promise,
который делает код более читаемым и линейным.

Event Loop — это механизм, который управляет выполнением асинхронного кода. Он проверяет стек вызовов (Call Stack) и, если стек пуст,
начинает выполнять асинхронные операции из очереди задач.
Благодаря этому происходит "неблокирующее" выполнение, и приложение не зависает при ожидании результата асинхронных операций.

3.
ООП (Объектно-ориентированное программирование) —  это подход к написанию кода, при котором программа рассматривается как набор объектов,
которые взаимодействуют друг с другом.

Ключевые принципы ООП:
Инкапсуляция: Скрытие внутренней реализации объекта и предоставление только необходимых интерфейсов для работы с ним.
Например, класс Car может иметь метод startEngine(), который включает двигатель, но никто не будет знать, как именно это происходит внутри.

Наследование: Возможность создавать новые классы на основе существующих, переняв их свойства и методы.
Например, можно создать класс ElectricCar, который наследует от класса Car, но добавляет свои особенности (например, зарядка вместо заправки).

Полиморфизм: Способность объектов разного типа иметь методы с одинаковыми именами, но разным поведением.
Например, метод move() может работать по-разному для Car и Bicycle.

Абстракция: Скрытие сложности и предоставление простого интерфейса.
Например, в классе Car можно скрыть сложную логику работы двигателя и дать пользователю простой метод для его запуска.

В JavaScript ООП реализовано через классы (введены в ES6).
Классы позволяют создавать объекты с помощью конструктора и наследовать методы от других классов.

4.
Когда пользователь вводит URL в адресную строку:

1) Браузер обращается к DNS-серверу для получения IP-адреса домена.

2) Браузер устанавливает соединение с сервером через протокол TCP/IP.
Для HTTPS-соединений также происходит установка защищенного соединения через TLS/SSL.

3) Браузер отправляет HTTP-запрос на сервер, запрашивая ресурс (например, HTML-документ).

4) Сервер отправляет ответ, который может содержать HTML, CSS, JavaScript и другие ресурсы (например, изображения).

5) Парсинг и рендеринг страницы:

Браузер начинает парсить HTML и строить DOM (Document Object Model) дерево.

Он парсит и применяет стили (CSS) и строит CSSOM (CSS Object Model).

Браузер объединяет DOM и CSSOM в Render Tree (дерево рендеринга).

Затем происходит paint (отрисовка) и compositing (составление изображения), и страница отображается.

6) Технологии для ускорения процесса:

Кэширование: Использование заголовков Cache-Control и Expires для хранения ресурсов локально, избегая повторной загрузки.

CDN : Использование CDN позволяет уменьшить время загрузки, так как ресурсы хранятся на серверах, расположенных ближе к пользователю.

Lazy loading: Ленивая загрузка позволяет загружать изображения и другие ресурсы только когда они становятся видимыми на экране.

Атрибуты async/defer для скриптов: Позволяют загружать и выполнять JavaScript, не блокируя рендеринг страницы.

Проблемы:

Безопасность: Один из главных рисков — это атаки типа Man-in-the-Middle (MITM), когда злоумышленник перехватывает данные,
передаваемые между клиентом и сервером.

CORS (Cross-Origin Resource Sharing): Когда сайт пытается сделать запрос к ресурсам с другого домена,
может возникнуть ошибка из-за политики безопасности браузеров (CORS), которая ограничивает междоменные запросы.
